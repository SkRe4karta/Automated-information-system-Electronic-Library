## Функциональные возможности

### Общие требования:
1. **Управление ролями**: 
   - `administrator` — имеет полный доступ (добавление, редактирование и удаление книг).
   - `moderator` — может редактировать книги и модерировать рецензии (для Варианта 1).
   - `user` — может оставлять рецензии.
2. **Аутентификация**: вход (`/login`), выход (`/logout`), хэширование паролей, запоминание сессий.
3. **Безопасность**:
   - Автоматический редирект неаутентифицированных пользователей с закрытых страниц на `/login` с флэш-сообщением.
   - Защита от несанкционированного доступа с выводом сообщения о недостатке прав.
   - Очистка ввода Markdown и текстовых полей от вредоносных тегов с помощью библиотеки `Bleach`.
4. **Управление обложками**:
   - Вычисление MD5-хэша обложки при загрузке.
   - Исключение дублирования файлов на диске (если файл с таким хэшем уже загружен, повторно на диск он не сохраняется, а новая книга привязывается к существующему файлу).
   - Безопасное именование файлов на сервере (UUID).
   - Автоматическое удаление файла с диска, если удаляется последняя книга, использовавшая эту обложку.
5. **Интеграция EasyMDE**:
   - Редактор Markdown подключен к описаниям книг и текстам рецензий.
   - При просмотре книги и рецензий Markdown компилируется в безопасный HTML.

## Индивидуальный вариант №1 — модерация рецензий

В проекте оставлен только первый индивидуальный вариант.

Реализовано:
- отдельная таблица статусов рецензий: `pending`, `approved`, `rejected`;
- при создании рецензия автоматически получает статус `pending` («На рассмотрении»);
- на странице книги отображаются только рецензии со статусом `approved`;
- пользователь видит раздел «Мои рецензии» со списком своих рецензий и их статусами;
- модератор и администратор видят раздел «Модерация рецензий»;
- список рецензий на модерации отсортирован по дате добавления и имеет пагинацию;
- на странице рассмотрения рецензии доступны действия «Одобрить» и «Отклонить».

## Установка и запуск

### 1. Создание виртуального окружения и установка зависимостей
Откройте терминал в папке проекта и выполните:
```powershell
# Создание виртуального окружения
python -m venv .venv

# Активация виртуального окружения (Windows)
.venv\Scripts\activate

# Установка библиотек
pip install -r requirements.txt
```

### 2. Запуск приложения
```powershell
python app.py
```
После запуска проект будет доступен в браузере по адресу: [http://127.0.0.1:5000/](http://127.0.0.1:5000/)

*Примечание: База данных `library.db` будет создана и наполнена тестовыми данными автоматически при первом старте.*
